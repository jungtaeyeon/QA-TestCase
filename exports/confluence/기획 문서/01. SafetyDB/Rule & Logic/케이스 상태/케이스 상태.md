# 개요

* 본 설계서는 사례(Case)의 인입부터 종결까지의 생애주기를 관리하는 ICSR 업무 단계(Workflow Stage)와 각 프로세스의 진행 상태(Status)를 정의.
* 모든 상태 명칭을 표준화하여 직관성을 높이고, 단계 간 전이 로직을 엄격히 제어하는 데 목적이 있음.

# **Workflow Stage (진행 단계 - Tracker 노출)**

사용자가 직접 확인하고 라우팅하는 업무의 물리적 위치입니다.

* Entry ➔ QC ➔ MA ➔ Approval(Approved) ➔ Submission ➔ Completion

# 통합 상태 관리 명세

모든 프로세스의 진행, 완료, 실패 상태는 In Progress, Success, Failed로 통일하여 표시 (몇가지의 예외사항이 있을 수 있음)

## 인입 및 생성 상태 (Import Status)

* 표시 위치: Convert/Import Status 화면 및 Tracker 내 관련 항목.
* In Progress (진행) 하위 단계:

  1. New Case Check: 중복 사례 판정
  2. Duplicate Check: 후속 보고 여부 판정
  3. Import: 제품 및 마스터 설정값 매핑
  4. Rule & Custom Apply: 이상사례 유형 자동 분류
  5. Case Creation
* Success (완료): 모든 인입 로직이 정상 완료되어 케이스 데이터 생성이 완료된 상태.
* Failed (실패): 인입 과정 중 오류가 발생하여 실패한 상태.

## 승인 상태 (Approval Status)

* In Progress: 승인 신청 후 수락 또는 반려를 대기하는 상태.
* Reject: 신청한 승인이 승인권자에 의해 반려된 상태. (사용자 혼선을 막기 위한 예외 명칭)

  + Reject일 경우, 담당자가 거절 시 돌아갈 단계를 선택해서 거절 가능. 선택한 단계로 돌아간다.
* Approved: 최종 승인 완료 단계로, 이후 보고 가능

## 보고 상태 (Submission Status)

* Ready to Submit: Approved 완료 후 보고 대기열에 올라온 상태. 이 상태에서만 Submit 버튼이 활성화
* In Progress (진행) 하위 단계:

  1. Queued: 전송 대기열 진입
  2. Transmitting: 데이터 송신 중
  3. Waiting ACK: 규제기관 응답 대기
  4. Received ACK: 응답 수신 완료
* Success (완료): 규제기관 접수 및 보고가 성공적으로 완료된 상태.
* Failed (실패) 하위 사유:

  + 오류: 시스템 또는 데이터 기술 오류
  + 타임아웃: 전송 시간 초과
  + 48시간 내 미수신: 보고 결과를 48시간 내에 받지 못한 경우
  + ACK Error: 수신된 응답 내 오류 포함

# 핵심 시스템 제어 로직

* 워크플로우 라우팅: 각 ICSR 단계는 사용자의 라우팅 행위에 의해 물리적으로 진행

### 보고 개시 조건

* 조건: 사례가 Ready to Submit 상태이고, 해당 규제기관의 Validation이 완료되었을 때 가능.

  + Ready to Submit 전환: 보고 Validation이 완료되면 해당 상태로 전환.
  + 실행: Ready to Submit 상태에서 Approved 되었을 때, Submit 버튼을 통해 보고를 진행.
  + Submit 버튼 클릭 시:

    - Ready to Submit+Approved일 때: 보고 진행
    - Approved 단계 아닐 경우: 안내문구 표시 후 보고 진행되니 않음.

### 사례 잠금 및 해제 (Lock & Unlock)

* 사례 잠금(Lock):

  + 자동 Lock: Approval 단계 진입 시 데이터 수정 기능이 자동으로 잠기며, 이후 단계(Approved 등)에서도 유지.
* 사례 잠금 해제(Unlock):

  + Case Edit 화면의 Unlock 클릭 시, 사용자가 직접 이전 단계(Entry/QC/MA 등)로 역라우팅하여 Unlock. Lock/Unlock은 Workflow Stage에 연동되며, Approved 미만 단계로 역라우팅.

### 완료 처리(Completion)

* Case Complete: Case Edit 상단의 버튼을 명시적으로 클릭해야만 단계가 Completion으로 변경.
