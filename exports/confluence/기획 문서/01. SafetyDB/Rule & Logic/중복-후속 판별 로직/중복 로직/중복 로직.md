# 개요

* 목적: 이미 처리된 케이스와 동일한 데이터가 중복 인입되는 것을 방지하여 업무 효율성을 높이고 데이터의 정확성을 확보함.
* 핵심 원칙: 인입되는 사례의 Source Type(국외/국내)에 따라 검증 프로세스를 이원화하여 판정함.

# 프로세싱 로직

시스템은 신규 케이스의 소스 유형에 따라 아래 두 가지 로직 중 하나를 적용

### [Logic A] XML

국외 보고는 C.1.8.1 번호를 일차적인 마스터 키로 사용하여 검증

1. C.1.8.1 검색: DB 내 일치하는 C.1.8.1 값이 있는지 검색

   * 일치 항목 없음: 결과 ➔ '신규(New)' 로 진행, Case ID 부여
2. 상세 필드 비교: 일치하는 C.1.8.1 발견 시, 제2절의 '주요 정보 필드' 전체를 대조

   1. 모든 필드 완전 동일: 결과 ➔ '중복(Duplicate)'.
   2. 주요 정보 상이: 결과 ➔ '후속(Follow-up)'

### [Logic B] Manual Intake

국내 수동 입력 사례는 필수 정보 필드의 전수 일치 여부를 중심으로 검증

* 시나리오 1: C.1.8.1 입력 값이 없을 때

  + 주요 정보 필드가 모두 일치하는 케이스 검색.
  + 일치 발견 시 ➔ '중복(Duplicate)' / 미발견 시 ➔ '신규(New)'.
  + 결과 출력 및 사용자 인터랙션

## 출력 데이터

* 검증 결과 : 신규 아님 ('중복(Duplicate)', '후속(Follow-up)'), '신규(New)' 중 하나를 반환
* 관련 케이스 ID (Related Case ID): 신규 아님('중복' 또는 '후속') 판정 시, 매칭된 기존 케이스의 ID를 함께 제공

## 사용자 컨펌 프로세스

* 중복(Duplicate) 및 후속 판정 시: 사용자에게 알림을 표시하고 '신규 사례로 진행'할지 '입력 취소 후 Tracker 이동'할지 최종 선택을 모달로 표시

  + ‘신규 사례로 진행’ 선택할 경우: Case ID 부여하고 새로운 케이스로 진행
  + ‘입력 취소 후 Tracker 이동’ 선택할 경우: 그대로 케이스 프로세싱 중지.

    - 후속 보고의 경우 Tracker에서 해당 Case ID에서 후속보고를 생성해 주세요 안내 문구 표시

### **트리거**:

* Logic A(XML): `C.1.8.1` 일치 항목 발견 시 (중복 또는 후속)
* Logic B(Manual): 주요 정보 필드 전수 일치 항목 발견 시 (중복)

### 모달 레이아웃 구조

#### 안내 문구:

* 입력하신 주요 정보가 아래의 기존 케이스와 일치합니다.
  매칭된 Case ID: [기존 Case ID]
  해당 케이스를 신규로 생성하시겠습니까?
  후속 보고를 생성할 경우, Tracker에서 해당 케이스에 후속보고를 생성해주세요.

#### 버튼:

|  |  |
| --- | --- |
| **버튼 명칭** | **액션 및 시스템 반응** |
| 신규로 케이스 생성하기 | 1. 현재 입력을 바탕으로 신규 Case ID를 부여함. 2. Intake 프로세스를 완료하고 Case Edit 화면으로 이동함. |
| 입력 취소 후 Tracker 이동 | 1. 현재 Intake 프로세스를 즉시 중단함(데이터 저장 안 함). 2. 모달을 닫고 이전 화면(Tracker)으로 복귀함. |
