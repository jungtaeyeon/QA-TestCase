# 프로덕트 불러오기

# 1. 개요

---

* **목적**: Case Edit 화면 중 '의약품(Drug)' 섹션의 식별 정보 영역(ICH E2B R3 G.k.1~G.k.3)의 필드 구성, 코딩 연동, 탭 생성 로직 및 프로덕트 불러오기 기능을 정의.
* **페이지 위치**: Safety > Tracker > Case Edit > Drug > 1

**핵심 설계 방향**:

* **탭 기반 다중 Drug**: Intake에서 선택한 의약품마다 별도 탭 생성. Study 연동 시 Arm/Blinding 정보에 따라 자동 탭 생성 [Release2].
* **프로덕트 불러오기**: WHODrug/Selected Product 팝업을 통한 코딩 및 마스터 연동.
* **Blinding 연동**: Study Configuration의 Blinding 값에 따라 맹검 의약품을 "Blinded IP of Study ID"로 자동 표시 [Release2].

# 4. 정책

---

## 4.1 WHODrug / MFDS Drug 팝업 & License Expire 규칙

의약품 코드 관련 필드(G.k.2.1.KR.1a, G.k.2.1.KR.1b)에 적용:

| 조건 | 동작 |
| --- | --- |
| License 유효 | WHODrug/MFDS Drug 팝업으로 입력 |
| License Expire | Free text로 전환하여 검증 |

## 4.2 Investigational Product Blinded(G.k.2.5) XML 전송 규칙

| 조건 | 동작 |
| --- | --- |
| Console > Company > Study Info > Blinding ≠ Open-Label AND C.5.Selta.1 Value = Blinded | True로 전송 |
| 그 외 | XML에 포함하지 않음 |

## 4.3 IDMP 미확정 필드

MPID(G.k.2.1.1a/b), PhPID(G.k.2.1.2a/b) 필드는 IDMP 확정 후 사용 예정.

## 4.4 Intake 연동 — 의약품명

| 조건 | 동작 |
| --- | --- |
| Intake & Triage에서 Product 입력됨 | G.k.2.2에 자동 입력 |

## 4.5 Study ID 연동 — 의약품명 표시 규칙 [Release 2]

| 조건 | 동작 |
| --- | --- |
| Study ID 연동 + Blind | "Blinded IP of Study ID Period Part N"로 표시 |
| Study ID 연동 + Open-label + WHODrug coding | Drug name 표시 |
| Selected Product coding | Regional Trade (Country/Dosage/Strength) 입력 |

## 4.6 Study ID 연동 — 의약품 코드 자동 입력 [Release 2]

| 조건 | 동작 |
| --- | --- |
| Study ID 연동 > Selected Product | Regional Drug ID 자동 입력 (G.k.2.1.KR.1b) |
| Study ID 연동 > WHODrug coding | Medicinalprod\_Id 자동 입력 (G.k.2.1.KR.1b) |
| Study ID 연동 > Product List coding | 의약품 코드 버전 (G.k.2.1.KR.1a) 빈 값 |
