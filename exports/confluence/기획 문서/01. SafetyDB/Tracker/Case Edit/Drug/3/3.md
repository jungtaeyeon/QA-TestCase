# 1. 개요

---

* **목적**: ICSR(개별 사례 안전성 보고서)의 의심/병용 약물 투여량 및 관련 정보(Dosage and Relevant Information, G.k.4.r)를 입력하고, 규제기관 보고를 위한 투여 이력 데이터를 생성함.
* **페이지 위치**: Tracker > Case Edit > Case > Drug > 3
* **핵심 로직**:

  + **값/단위 쌍의 조건부 필수**: 수치 필드(투여량/투여간격/투여기간) 입력 시 해당 단위 필드가 조건부 필수로 활성화되며, 단위 필드는 ICH 허용 UCUM 코드만 허용.
  + **날짜 선후 관계 정합성**: 투여 시작일(G.k.4.r.4)과 종료일(G.k.4.r.5)의 선후 관계 검증 및 미래 일자 입력 차단.

# 3. 기능 설명

---

| # | 기능명 | 설명 | 액션 |
| --- | --- | --- | --- |
| 1 | 값/단위 쌍 검증 | Dose number↔unit, Interval number↔unit, Duration number↔unit — 수치 입력 시 단위 필수 | 저장/Validation |
| 2 | 투여간격 공란 처리 | G.k.4.r.3이 {cyclical}/{asnecessary}/{total}인 경우 G.k.4.r.2를 공란으로 처리 | G.k.4.r.3 값 변경 시 |
| 3 | 날짜 선후 검증 | G.k.4.r.5 ≥ G.k.4.r.4 검증. 위반 시 저장 차단 + 에러 메시지 | 저장 시 |
| 4 | Product 연동 자동 입력 | Product 코딩 연동 시 Dosage Formulation Term을 G.k.4.r.9.1에 자동 입력 | Product 연동 시 |

# 4. 정책

---

## 4.1 값/단위 쌍 조건부 필수

수치 필드 입력 시 해당 단위 필드가 조건부 필수로 활성화:

| 수치 필드 | 단위 필드 | 허용 코드 |
| --- | --- | --- |
| Dose (number) G.k.4.r.1a | Dose (unit) G.k.4.r.1b | ICH 허용 UCUM 코드 |
| Number of Units in the Interval G.k.4.r.2 | Definition of the Time Interval Unit G.k.4.r.3 | UCUM + {cyclical}/{asnecessary}/{total} |
| Duration (number) G.k.4.r.6a | Duration (unit) G.k.4.r.6b | 10.a, a, mo, wk, d, h, min, s |

## 4.2 투여간격 특수 단위 처리

| 조건 | 동작 |
| --- | --- |
| G.k.4.r.3 = {cyclical}, {asnecessary}, 또는 {total} | G.k.4.r.2(투여간격) 공란으로 처리 |

## 4.3 투여 날짜 선후 관계 검증

| 규칙 | 동작 |
| --- | --- |
| G.k.4.r.5(종료일) < G.k.4.r.4(시작일) | 저장 차단, 에러 메시지: "종료일은 시작일보다 이전일 수 없습니다" |
| 최소 정밀도 | CCYY(연도)까지 입력 필수 |
| 미래 일자 입력 | 차단 (현재 일시 기준) |
| 날짜 미상 | nullFlavor(MSK/ASKU/NASK)로 대체 가능 |

## 4.6 Product 연동 — 제형 자동 입력

| 조건 | 동작 |
| --- | --- |
| Product 연동 시 | Dosage Formulation의 Term을 G.k.4.r.9.1에 자동 입력 |
| 라이센스 미연결 시 | 자유 기재 |
