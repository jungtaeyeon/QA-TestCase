# 1. 개요

---

**목적**: Case Edit 화면 중 '의약품(Drug)' 섹션의 성분 정보 영역(ICH E2B R3 G.k.2.3.r)의 필드 구성 및 코딩 연동을 정의. 의약품을 구성하는 유효성분(Active Pharmaceutical Ingredient)의 성분명, 용량, 코딩 정보를 구조화하여 규제기관 보고 및 Signal Detection에 활용할 수 있는 데이터를 확보.

**페이지 위치**: Safety > Tracker > Case Edit > Drug > 2

**핵심 설계 방향**:

* **반복 구조(r)**: 하나의 의약품에 다수의 성분(G.k.2.3.r)을 등록 가능하며, 각 성분별로 독립적인 코딩 및 용량 정보를 관리.
* **Study 연동 [Release2]**: Study Configuration에서 Selected Product 코딩 시 API Name, WHODrug 코딩 시 Substance name이 자동 입력.

# 4. 정책

---

## 4.1 Study ID 연동 — 성분명 자동 입력 [Release2 예정]

| 조건 | 동작 |
| --- | --- |
| Study ID 연동 > Selected Product Coding | API 유효성분 테이블의 API Name을 G.k.2.3.r.1에 자동 입력 |
| Study ID 연동 > WHODrug Coding | Substance name을 G.k.2.3.r.1에 자동 입력 |

## 4.2 Strength (unit) — UCUM 코드 제한

| 조건 | 동작 |
| --- | --- |
| Strength (number) 입력됨 | Strength (unit)은 UCUM 코드로 허용된 코드만 입력 가능 |
