# 1. 개요

본 설계서는 사례(Case) 처리의 효율성과 투명성을 확보하기 위해 역할 기반의 자동 이송(Routing), 타임라인 기반의 협업, 승인 연동형 데이터 잠금(Lock) 로직을 정의합니다.

---

# 2. 워크플로우 단계 정의 (Workflow Stages)

업무는 아래의 단계를 순차적으로 거치며, 각 단계 전환은 사용자의 Routing(Forward/Reject) 액션에 의해 발생합니다.

|  |  |  |
| --- | --- | --- |
| 단계 | 주요 역할 | 단계 설명 |
| Entry | 입력자 | 사례 데이터의 최초 입력 및 기초 정보 구성 |
| QC | 리뷰어 | 입력 데이터의 품질 검수 및 정합성 확인 |
| MA | 의학적 평가자 | 의학적 인과성 평가 및 전문 의견 작성 |
| Approval | 승인권자 | 보고서 최종 승인 (세부 상태: In Progress, Reject, Approved) |
| Completion | 시스템/사용자 | 모든 업무가 완료된 종결 단계 (Case Complete 시 전환) |

# 3. 통합 커뮤니케이션 패널

Case Edit 화면 우측에 위치하며, 협업을 위한 '채팅(Chat)'과 업무 이송을 위한 **'라우팅(Routing)'** 기능이 통합된 영역

### **3.1. UI/UX 구성**

* **위치**: Case Edit 화면 우측 레이아웃에 배치
* **가시성 제어**: 기본적으로 숨김(Collapsed) 상태이며, 우측 상단의 '패널 열기/접기' 버튼 클릭 시 영역이 노출
* **탭 구조**: 입력창의 탭을 통해 [코멘트(Chat)]와 **[라우팅(Workflow)]** 기능을 전환하며 사용

### **3.2. 채팅형 코멘트 기능 (Chat Room Style)**

* **목적**: 사례 데이터의 각 필드에 대한 질의응답 및 협업 내용을 기록하고 공유
* **기록 항목**: 모든 메시지는 작성자 이름, 작성 시간(YYYY-MM-DD HH:MM:SS), 메시지 내용을 포함
* **상호작용**: 채팅방 형태의 UI로 구성되어 시계열 순으로 대화 내용이 나열되며, 다른 담당자가 실시간으로 확인 가

### **3.3. 워크플로우 라우팅 기능**

* **순방향 로직**: Entry > QC > MA > Approval > Completion 순으로 진행
* **조작 항목**:

  + **단계 선택**: 이동 가능한 단계(순방향/양방향)를 선택합니다.

    - 순방향: Approval > Completion
    - 양방향 가능: Entry <> QC <> MA <> Approval
  + **담당자 지정**: 해당 Workspace의 App에 접근 허용된 사용자를 드롭다운으로 선택.
  + **사유 입력**: 이송/반려 사유를 입력합니다. (입력 시 해당 사유는 통합 타임라인에 자동 기록됩니다.)
* **권한 이전**: Forward 클릭 시 사례의 '편집 권한(Assignee)'이 새 담당자에게 즉시 이월되며, 기존 담당자는 Read-only로 전환

# 4. 타임라인 기반 업무 협업 (Timeline Collaboration)

업무의 이력을 시계열로 관리하여 의사결정 과정을 투명하게 기록

## 4.1. Forward/Reject 연동 코멘트

* 의무 기록: 다음 단계로 이송(Forward)하거나 이전 단계로 반려(Reject)할 경우, 사유 코멘트 작성을 필수화
* 자동 기록: 작성된 코멘트는 해당 액션(누가, 언제, 어디로)과 함께 타임라인에 즉시 저장

## 4.2 기록되는 사항

* 단계 변경 시
* 코멘트트 생성 시
* 저장 이력: 변경 이력은 보이지 않고, 저장된 이력만 표시: ‘변경사항이 저장되었습니다.’ 문구와 함께 저장자 이름, 저장 시간(YYYY-MM-DD HH:MM:SS)
* Lock/Unlock 사항 표시:

  + 잠금 시 : ‘사례가 잠금되었습니다. 사례를 더이상 수정할 수 없습니다.’ 문구와 잠금 주체자, 시간(YYYY-MM-DD HH:MM:SS)
  + 잠금 해제 시 : 잠금 시 : ‘사례가 잠금 해제되었습니다.’ 문구와 잠금 주체자, 시간(YYYY-MM-DD HH:MM:SS)

## 4.3. 통합 타임라인

* System Log: 상태 변경, 승인/반려, 단계 이동 등 시스템 이벤트 자동 기록.
* User Comment: 사용자가 직접 입력한 의견 및 사유 기록.
* 표시 방식: 모든 기록은 시계열 순으로 나열되어 사례의 전체 히스토리를 한눈에 파악할 수 있도록 제공.

# 5. 승인 연동형 자동 잠금

최종 보고 데이터의 무결성을 보호하기 위한 강제 제어 로직입니다.

### 5.1. 최종 승인 시 자동 잠금

* 로직: Approval 단계에서 최종 'Approved' 상태로 처리가 완료되면, 해당 사례는 즉시 Read-only(Case Lock) 상태로 전환.
* 효과: 승인된 데이터는 어떠한 수동 수정도 불가능한 상태.

### 5.2. 잠금 해제(Unlock) 및 재승인 로직

* 승인 효력 무효화: Locked 사례의 데이터를 수정하기 위해 Unlock을 실행할 경우, 기존에 완료된 승인(Approved) 효력은 시스템에 의해 자동으로 무효화(Invalidated) 처리.

  + 해당 단계는 자동으로 MA 단계로 변환. 수정 가능
* 수정 사유 필수화: Unlock 시 '수정 사유(Reason for Change)' 입력을 필수화하며, 이는 타임라인에 기록.
* 정합성 강제: 데이터 수정 후에는 반드시 다시 워크플로우 초기 단계 혹은 필요한 단계를 거쳐 재승인(Re-approval)을 받도록 시스템적으로 강제.
