# **1. 홈 화면 설계**

플랫폼의 홈 화면은 Org 홈과 Workspace 홈으로 구분된다. Org 홈은 전 역할의 공통 진입점이며, Workspace 홈은 Workspace 범위의 통합 진입점 역할을 한다. 앱 기능은 Workspace 홈 내에서 사이드바를 통해 전환하여 사용한다.

## **1.1 홈 화면 구성요소**

Org 홈과 Workspace 홈의 구성요소는 계층에 따라 다르게 구성된다.

| 구성요소 | Org 홈 데이터 범위 | Workspace 홈 데이터 범위 |
| --- | --- | --- |
| 공지사항 | 전체 Org 공지. 최근 5건 표시. [더보기] 클릭 시 공지사항 목록으로 이동 | Org 홈 공지와 동일하며, Workspace 단위 내부 공지 추가 표시 |
| My Tasks: 내가 오너인 케이스 | 전체 Workspace 통합 기준 내가 Owner 인 케이스 중 최근 업데이트 순 5건 | 제공하지 않음 |
| My Task: 나에게 배정된 케이스 | 전체 Workspace 통합 기준 내가 Assignee로 배정된 건 중 최근 업데이트순 최대 5건 | 제공하지 않음 |
| 다가오는 보고 마감일 | 전체 Workspace 내 내가 Owner인 케이스 중 보고 대상 건의 마감일 임박순 최대 10건 | 해당 Workspace 내 내가 Owner인 케이스 중 보고 대상 건의 마감일 임박순 최대 10건 |
| 바로가기 | Workspace 이름 목록 표시 | 제공하지 않음 |

> Workspace 홈은 앱 선택 화면을 대체하며, 사이드바에서 앱 간 자유 전환이 가능하다.

> 앱별 별도 홈 화면은 존재하지 않으며 Workspace 홈이 통합 진입점 역할을 한다.

# **1.2 Org 홈**

전 역할이 로그인 후 최초 진입하는 화면이다. Org 단위의 통합 현황 정보를 제공하고, Workspace 진입 경로를 포함한다.

## **접근 대상**

* 전 역할이 접근 가능

## **역할 정의**

* Org 홈은 Org 단위의 관리 및 통합 조회를 위한 공통 진입 화면이다.
* 전체 Workspace 통합 현황, Org 수준 메뉴, 관리자 기능 진입을 목적으로 한다.
* 전 역할은 Org 홈을 공통 진입점으로 사용하며, 이후 Workspace 홈과 앱 화면으로 이동한다.

## **사이드바 메뉴 구성**

Org 홈에서 표시되는 사이드바는 앱 내 사이드바와 별개이며, Org 수준 기능에 대한 진입점을 제공하며, 역할에 따라 진입할 수 있는 메뉴가 다르다.

| 메뉴 | 이동 대상 | 접근 가능 역할 | 비고 | 일정 |
| --- | --- | --- | --- | --- |
| 홈 (Home) | Org 홈 메인 화면 | 전 역할 |  | 최우선 |
| 공지사항 (Notice) | 공지사항 목록 화면, 전체 Org 공지 목록 표시 | 전 역할 |  | 고도화 |
| 활동 기록 (Activity Log) | Org Activity Log 화면 | 전 역할 |  | 최우선 |
| 관리자 (Admin) | Org Admin 화면  하위 항목: 사용자 관리 / 계약·플랜 / 연결 서비스 / 감사 로그 | Master Admin / Org Admin/ Viewer |  | 최우선 |

참고: 역할별 세부 권한 범위는 Account 정책 문서를 따른다.

* *\* 추후 추가 검토 항목 (v1 범위 외)*

  + *대시보드: 현재 케이스의 상태와 퀄리티를 확인할 수 있는 데이터 시각화 페이지*

    - *Status*

  + *Signal Detection (안전성 시그널 요약): 탐지된 약물 신호 및 분석 현황. Signal Detection 기획 시 Org 홈 메뉴로 추가 검토.*
  + *Regulatory Calendar (규제 보고 캘린더): PSUR·PBRER·DSUR 등 정기 보고서 제출 마감일 관리. 개별 ICSR 마감과 별개로 제품·성분 단위 일정 관리 필요. Safety DB 기획 범위 확정 후 검토.*

    - [IDEA] 추후에 Calendar 넣을 때, 공공데이터포탈에서 의약품 유효기간 정보를 받아서 해당 조직 제품의 유효기간을 표시해주는 것도 좋을 것 같습니다.
  + *제품 안전성 모니터링 (Product Safety Monitoring): 제품 단위 안전성 정보를 통합하여 Org 수준에서 조회하는 기능군. 세부 구성 요소는 다음과 같으며, 별도 기획 후 단일 메뉴 또는 하위 탭 구조로 편입 여부를 결정한다.*

    - *제품별 안전성 정보 통합 대시보드 (Product-Specific Safety Dashboard): 제품·성분 단위 이상사례 집계, 심각도 분포, 보고 현황 등 안전성 지표 시각화*
    - *안전성 데이터 흐름 시각화 (Safety Data Flow): 수집 → 변환 → 검토 → 보고 각 단계별 케이스 흐름 및 병목 지점 시각화*
    - *규제 일정 관리 (Regulatory Schedule Management): 제품·적응증 단위 정기 보고 일정(PSUR/PBRER/DSUR 등) 등록 및 마감일 추적. Regulatory Calendar와 연동 검토*

# **1.3 Workspace 홈: 워크스페이스 홈은 고도화 일정으로 진행(**[https://selta.atlassian.net/wiki/spaces/IS/pages/25067537/Home+Navigation#%EC%9D%BC%EC%A0%95-%EA%B5%AC%EB%B6%84%3A-%EC%B5%9C%EC%9A%B0%EC%84%A0%EC%9C%BC%EB%A1%9C-%ED%99%88%EC%97%90%EC%84%9C%EB%8A%94-%EC%9B%8C%ED%81%AC%EC%8A%A4%ED%8E%98%EC%9D%B4%EC%8A%A4-%EC%9D%B4%EB%8F%99%EB%A7%8C-%EB%B3%B4%EC%97%AC%EC%A4%80%EB%8B%A4.-%EC%95%B1%ED%99%88%EC%9D%80-%ED%8A%B8%EB%9E%98%EC%BB%A4%EB%8F%84-%EB%8C%80%EC%B2%B4%ED%95%9C%EB%8B%A4.](https://selta.atlassian.net/wiki/spaces/IS/pages/25067537/Home%2BNavigation#%EC%9D%BC%EC%A0%95-%EA%B5%AC%EB%B6%84%3A-%EC%B5%9C%EC%9A%B0%EC%84%A0%EC%9C%BC%EB%A1%9C-%ED%99%88%EC%97%90%EC%84%9C%EB%8A%94-%EC%9B%8C%ED%81%AC%EC%8A%A4%ED%8E%98%EC%9D%B4%EC%8A%A4-%EC%9D%B4%EB%8F%99%EB%A7%8C-%EB%B3%B4%EC%97%AC%EC%A4%80%EB%8B%A4.-%EC%95%B1%ED%99%88%EC%9D%80-%ED%8A%B8%EB%9E%98%EC%BB%A4%EB%8F%84-%EB%8C%80%EC%B2%B4%ED%95%9C%EB%8B%A4.) )

Workspace 홈은 사용자가 Org 홈에서 Workspace를 선택한 후 진입하는 화면이다. 앱 간 전환의 기준점이 되며, Workspace 범위의 현황 정보를 제공한다.

## **접근 조건**

* 전 역할. Org 홈에서 Workspace 선택 후 진입.
* Workspace URL 직접 입력 시에도 진입 가능 (로그인 후 해당 Workspace 홈으로 진입).

## **역할 정의**

* Workspace 단위의 현황 및 공지와 현재 워크스페이스에 있는 앱을 제공한다.
* 앱 간 전환은 사이드바에서 처리한다.
* 노출 목적으로 미구독 앱이 구독 중인 앱과 함께 사이드바에 표시될 수 있다.

# **2. 내비게이션 구조**

본 섹션은 홈 화면을 기반으로 사용자가 서비스 전체를 이동하는 방식을 정의한다. 전체 레이아웃 골격, 역할별 진입 경로, 글로벌 헤더, 앱 내 사이드바, 어드민 진입 경로를 포함한다.

## **2.1 전체 레이아웃 골격**

* 글로벌 헤더는 모든 화면에서 고정 표시된다.
* 사이드바는 Org 홈 및 Workspace 홈, 앱 내부 화면에서 표시된다.

## **2.2 역할별 로그인 후 진입 경로**

로그인 완료 후 진입 경로는 전 역할이 Org 홈으로 먼저 진입한다. Workspace 선택은 Org 홈 내 Workspace 바로가기 영역에서 수행한다.

### 사전 상태 확인 규칙

로그인 완료 후 역할별 진입 경로를 적용하기 전에 Org 상태를 먼저 확인한다.

* Pre-Active: 고객 로그인 차단, Org 홈/Workspace 홈 및 앱 화면 모드 직접 진입 불가
* Active: 정상 진입 흐름 진행
* Terminated: Org 홈, Workspace 홈, 앱 화면 모두 직접 진입 불가.

### **2.2.1 전 역할 공통 진입 흐름**

로그인 완료 후 전 역할은 Org 홈으로 진입한다. Org 홈에서 Workspace를 선택하면 새 탭으로 Workspace 홈이 열린다. 이후 앱 간 전환은 사이드바에서 처리한다.

| 조건 | 진입 경로 |
| --- | --- |
| Workspace 1개 | Org 홈 → 해당 Workspace 홈 (새 탭) |
| Workspace 2개 이상 | Org 홈 내 Workspace 선택 → 선택한 Workspace 홈 (새 탭) |

역할별 차이는 진입 경로가 아니라 Org 홈/Workspace 홈/앱 화면에서 표시되는 메뉴와 데이터 범위로 구분된다.

### **📋 공통 진입 흐름**

| 단계 | 내용 |
| --- | --- |
| ① | 로그인 완료 (MFA 포함) → Org 홈 진입 |
| ② | Org 홈에서 Workspace 선택 → 새 탭으로 Workspace 홈 오픈 |
| ③ | 사이드바에서 앱 선택 → 해당 앱 메뉴로 노출 후 메뉴 클릭 시 해당 화면으로 이동 |
| ④ | 앱 탭 닫기 → Org 홈 탭 유지 |

### **2.2.2 역할별 표시 범위**

|  |  |  |  |
| --- | --- | --- | --- |

| Org Admin | Org 홈 전체 구성 + 관리자 메뉴 | (권한 부여 시) 해당 WS 현황 | (권한 부여 시) 권한 부여된 앱 |
| Viewer | Org 홈 전체 구성 + 관리자 메뉴 (읽기 전용) | (권한 부여 시) 해당 WS 현황 (읽기 전용) | (권한 부여 시) 권한 부여된 앱 (읽기 전용) |

## **2.4 앱 내 내비게이션 (사이드바)**

앱 내부 화면에서 표시되며, 현재 앱의 메뉴 내비게이션과 앱 전환 기능을 제공한다. Org 홈의 사이드바는 1.2를 참조한다.

| 요소 | 설명 |
| --- | --- |
| 앱 목록 | 현재 워크스페이스에서 사용 가능한 앱과 (노출을 목적으로 한) 사용하지 않는 앱을 노출. 구독 중인 앱은 정상 표시. 미구독 앱은 구독 중인 앱 하단에 표시. 앱 선택 시 아래 앱별 메뉴가 해당 앱 메뉴로 전환. |
| 앱별 메뉴 | 현재 앱의 기능 메뉴. 역할 및 권한에 따라 노출 항목 분기 |
| Admin | App Admin 역할에게만 노출 |

* 앱 전환 전 미저장 데이터가 있는 경우 경고 모달을 선행 표시한다.

### **미구독 앱 노출 정책**

* 사이드바 앱 노출 영역에 미구독 앱 노출 영역을 둘 수 있다.
* 미구독앱 노출 기준: 현재 백오피스에서 계약 시 활성화 할 수 있는 앱 중 해당 고객이 계약하지 않은 앱을 노출
* (현재는 앱이 많이 없어서 일단 노출할 수 있는 것은 다 노출하되, 특정 시점에는 미구독앱을 노출 할 것을 지정하는 기능 개발이 필요)
* 해당 영역은 앱 전환 메뉴와 분리된 보조 영역으로 구성한다.
* 미구독 앱은 클릭 시 1차로 현재 탭에서 새 탭을 띄워 회사 홈페이지의 앱 소개 화면으로 이동시킨다.
* 워크스페이스에 구독 중인 앱이 없고 미구독 앱만 있는 경우: 해당 상황은 있을 수가 없으나, 만약 이렇게 된다면 사이드바에 "사용 가능한 앱이 없습니다. 계약을 진행해주세요" 안내 텍스트 노출

### **📋 앱 전환 흐름**

| 단계 | 내용 |
| --- | --- |
| ① | 사이드바 앱 목록에서 전환할 앱 선택 |
| ② | 해당 앱의 메뉴 노출 / 화면은 변경 없이 현재 화면 상태 유지 이후 이동할 메뉴 항목 클릭  ex. Safety앱 사용중이다가 Litus로 전환한다 했을 때 Safety화면은 유지한채 Litus 앱 메뉴가 노출 |
| ③ | 미저장 데이터 감지 시: "저장하지 않은 변경 사항이 있습니다. 계속 하시겠습니까?" 경고 모달 표시 |
| ④ | 경고 모달 — [계속] 클릭: 현재 탭에서 선택한 앱 화면으로 이동. 미저장 데이터 폐기 |
| ⑤ | 경고 모달 — [취소] 클릭: 모달 닫힘. 현재 앱 화면 유지 |
| ⑥ | 미저장 데이터 없는 경우: 경고 모달 없이 현재 탭에서 선택 앱 화면으로 이동 |

### **탭 동작 원칙**

* Org 홈에서 Workspace 홈으로의 진입은 새 탭으로 처리한다.
* 앱 내부 이동 및 앱 간 전환은 현재 탭에서 이동하는 방식으로 처리한다.

## **2.5 URL 접근 및 딥링크 정책**

### **딥링크 직접 진입**

북마크 또는 자동 완성되는 특정 URL에 직접 접근하는 경우 아래 규칙을 따르되, 구체적인 동작은 별도 개발 설계에 따른다.

|  |  |
| --- | --- |
| 상황 | 처리 |
| 로그인 O + 권한 O | 해당 URL로 직접 진입 |
| 로그인 O + 접근 권한 없음. | 접근 불가 안내(해당 URL로는 접근 할 수 없습니다. 자세한 사항은 관리자에게 문의하세요.) + 5초 카운트 후 Org 홈 리다이렉트 |
| 로그인 X | 로그인 화면 표시 → 로그인 완료 후 원래 URL 복원 시도 |
| 로그인 X + 로그인 후 권한 없음 | 로그인 완료 후 접근 불가 안내 + 5초 후 Org 홈 리다이렉트 |

여기서 권한은 Org 상태, Workspace 접근 가능 여부, 앱 접근 권한을 포함한다.

만약 북마크 또는 자동 완성되는 특정 URL이 존재하지 않게 되는 경우(=존재하지 않는 URL에 접근하는 경우에도)도 접근 불가 안내 + 5초 후 Org 홈 리다이렉트

\*참고: URL 구조 및 기술적 설계는 본 문서 범위에 포함하지 않으며, 별도 개발 설계를 따른다.

### **Workspace URL 직접 로그인**

Workspace URL로 직접 접근 시 로그인 화면을 표시하고, 로그인 완료 후 해당 Workspace 홈으로 진입한다.

### **세션 만료 후 재접근**

세션 만료 후 재로그인 시 Org 홈으로 새로 진입한다. 만료 전 페이지로 복원하지 않는다.

## **2.6 세션 및 탭 정책**

### **탭 및 창 정의**

* 탭(Tab): 동일 브라우저 창 안에서 열린 페이지 단위
* 창(Window): 별도로 실행된 브라우저 창

본 문서에서 "새 탭"은 현재 화면과 분리된 별도 진입 정책을 의미한다. Cookie 기반 인증 구조에서 동일 브라우저 내 탭과 창은 동일 세션을 공유한다. Workspace 홈은 Org 홈에서 별도 탭으로 연다. 동일 Workspace의 기존 탭 재사용 여부 및 자동 포커스 이동 등은 본 문서에서 보장하지 않으며, 세부 구현은 개발 설계를 따른다.

### **세션 공유 원칙**

동일 브라우저 내 모든 탭과 창은 하나의 세션을 공유한다.

| 항목 | 규칙 |
| --- | --- |
| 멀티탭 | 동일 브라우저 내 탭과 창은 하나의 세션을 공유한다. 세션 충돌 없음. 마지막 액션 기준으로 타임아웃 갱신 |
| 탭 간 세션 | Org 홈 탭과 Workspace/앱 탭은 세션 공유 |
| 로그아웃 | 어느 탭에서 로그아웃해도 동일 브라우저 내 전체 탭/창 로그아웃    로그아웃 시 동작   1. Org홈에서 로그아웃: 로그인 화면 노출 2. 워크스페이스 홈 및 앱에서 로그아웃: 탭 닫기 |
| 탭 닫기 | 미저장 데이터 있는 경우 경고 표시 |
| 세션 타임아웃 | 세션 유효시간 및 만료 동작은 ACC 정책을 따른다 |

## **2.7 기본 어드민 진입 경로**

| 어드민 유형 | 진입 경로 | 접근 가능 역할 |
| --- | --- | --- |
| Org 어드민 | Org 홈 내 [관리자 (Admin)] 메뉴 | Master Admin / Org Admin/ Viewer |
| App 어드민 | 워크스페이스 홈 -> 앱 내 사이드바 관리자 | App Admin |

### 참고: Break-Glass 예외 접근

* Break-Glass 접근이 허용되면, 해당 Backoffice Admin은 대상 Org 범위 내 모든 Workspace와 App에 접근할 수 있다.
* 이 접근은 기존 역할 변경이 아니라, 시간 제한이 있는 예외 접근 세션으로 처리한다.
* Break-Glass 접근 범위 및 권한 상세는 BOF/AUD/MT를 참조한다.
