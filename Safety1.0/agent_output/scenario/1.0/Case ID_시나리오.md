### C. 번호 조합 — Year

#### SC-CASEID-005 — Year 옵션 3종 설정 및 Case ID 반영

| 관점 | 설계 기법 | 우선순위 | 관련 REQ |
|------|---------|---------|---------|
| Functional | EP (3종 옵션) | High | REQ-009, REQ-010, REQ-011 |

**목적**: Year 설정(2-digit / 4-digit / N/A)에 따라 Case ID 내 연도 표기와 초기화 정책이 올바르게 동작함을 확인한다.

**사전 조건**: C.1.4가 설정된 케이스 생성 환경

**검증 포인트**:
1. Year = 2-digit(YY) → 케이스 생성 시 연도 뒤 2자리 표기 (예: `26XXXXX`)
2. Year = 4-digit(YYYY) → 연도 전체 4자리 표기 (예: `2026XXXXX`)
3. Year = N/A → 연도 미표기, 누적 증가 확인
4. 2-digit/4-digit 설정 시: 1월 1일 00:00:00 기준 일련번호 리셋 정책 확인
5. N/A 설정 시: 연도 변경 후에도 일련번호 초기화 없이 누적 확인

#### SC-CASEID-013 — 자릿수 중간 변경: 이후 케이스 적용 / 기존 케이스 유지

| 관점 | 설계 기법 | 우선순위 | 관련 REQ |
|------|---------|---------|---------|
