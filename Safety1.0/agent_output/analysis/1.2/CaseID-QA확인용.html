Year 설정(2-digit / 4-digit / N/A)의 최초 진입 시 기본 선택값이 기획서 본문에 명시되지 않았습니다. QA는 2-digit이 최상단 값이므로 기본값으로 추정하여 TC를 작성합니다.

MISSING-002 🟡 누락 영향도: High Tier-1 ⛔ TC 작성 보류

연도 설정별 채번 방식만 정의됨. 운영 중 Year 설정 변경 시 처리 방식 없음.
운영 중 Year 설정을 변경할 경우(예: 2-digit → N/A), 이미 생성된 케이스의 Case ID 표시 및 번호 연속성 처리 방식이 정의되지 않았습니다. 특히 2-digit → N/A 전환 시 리셋 여부와 번호 누적 기준이 불명확합니다.
권장 조치: "Year 설정 변경 시 기존 케이스는 변경 전 자릿수/연도 형식 그대로 유지, 이후 생성 케이스부터 새 설정 적용" 형태로 명시 요청.

MISSING-004 🟡 누락 영향도: High Tier-1 ⛔ TC 작성 보류
