🔴 기획 충돌 (Conflict) — 1건
CONFLICT-001 🔴 충돌 영향도: High ⛔ TC 작성 보류
Void/Nullify 기능의 현재/추후 범위 충돌
문서 A §2.1 사례 생애주기 관리
"수정, 삭제(Void/Nullify), 보관(Archive) 기능 제공 [추후 예정]"
문서 B §4.1 Tracker 그리드 — FU No 컬럼 비고
"수정/무효화 시 Amend/Null 표시, 후속보고 순번은 그대로 동일하게 부여"
필터링 옵션: "Null, Amend"
§2.1에서 Void/Nullify는 [추후 예정]으로 명시되어 현재 구현 범위 밖입니다. 그러나 §4.1 FU No 컬럼에는 "무효화 시 Null 표시" 로직이 현재 기능처럼 기술되어 있고, 필터링 옵션에도 "Null"이 포함되어 있습니다. 이 FU No의 Null 표시가 현재 범위인지 추후 범위인지 불명확합니다.
권장 조치: Void/Nullify 기능의 현재/추후 범위를 확정하고, FU No Null 표시 로직의 구현 시점을 기획서에 명시해주세요. 현재 범위이면 §2.1에서 FU No 로직은 현재 포함 항목으로 분리해야 합니다.

Date Range 필터 RA Due 기준 날짜 범위, Day0 기준 날짜 범위, 두 필터 동시 적용 Medium BVA, DT FunctionalBoundary
컬럼 필터링 각 컬럼별 필터링 옵션 동작, 다중 필터 조합, 필터 초기화 Medium EP, DT FunctionalNegative
⚡ 엣지 케이스

경계값 / 상태 전이

  • RA Due: D-Day 당일 (Today = D-Day) 표시값 정확성
  • RA Due: Overdue이지만 Submission Date가 D-Day와 동일한 날인 경우 (On Time 경계)
  • RA Due: Submission Date가 없고 D-Day 초과인 경우 → Missing 처리
  • FU No: 동일 원본 케이스에서 FU를 연속으로 2회 이상 생성 시 번호 순서
  • Country: C.2.r.3과 E.i.9 모두 없는 케이스의 Country 컬럼 표시
  • Report Type: C.1.3이 4(Unknown)인 케이스의 표시값

권한 / 동시성

  • 두 사용자가 동일 케이스를 동시에 Edit 요청하는 경우
  • Case Edit 중 다른 사용자가 같은 케이스의 FU 생성을 시도하는 경우
  • 검색 결과가 0건인 상태에서 필터 추가 적용

외부 연동

  • XML Import 도중 Data-hub 화면 이동 실패 시 Tracker 상태
  • AE Type 분류 로직 실행 실패 시 컬럼 표시값 (Unclassifiable? 빈값?)
  • MedDRA coded Term이 없는 케이스의 해당 컬럼 표시

①정상Tracker 접속 → RA Due 필터에서 "Day-N" 선택 → Day-1~Day-N 케이스만 필터링 → AE Type·Assignee 확인 → 긴급 케이스 우선 처리
②대안Date Range (RA Due) 필터와 AE Type 필터 동시 적용 → SUSAR 케이스만 마감 기준으로 필터링
③오류RA Due = Missing인 케이스 다수 존재 → Submission Date 누락 케이스 일괄 파악
