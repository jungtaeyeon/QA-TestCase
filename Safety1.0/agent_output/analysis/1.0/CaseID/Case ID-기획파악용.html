🔢 번호 조합 구성요소 설정

① Year (연도 표시 및 리셋 정책)

참조 기준: 최초 발생인지일 (C.1.4)의 데이터값

구분표기 방식번호 초기화비고
2-digit yearC.1.4 연도 뒤 두 자리 YY (예: 26)매년 1월 1일 00:00:00 리셋연간 누적 채번
4-digit yearC.1.4 연도 전체 네 자리 YYYY (예: 2026)매년 1월 1일 00:00:00 리셋연간 누적 채번
N/A연도 미표기초기화 없이 누적 증가전체 누적 채번
툴팁 문구 (Year 설정란 옆):
"연도 선택 시 매년 초, 연월 선택 시 매월 초에 일련번호가 '00001'로 초기화됩니다. N/A 선택 시(Chip을 추가하지 않았을 경우) 초기화 없이 번호가 계속 누적됩니다."

🔢 일련번호 자릿수 및 시작 번호 설정

일련번호 자릿수 설정

항목내용
선택값3, 4, 5, 6, 7, 8, 9 (최대 9자리)
Zero-padding선택한 자릿수만큼 번호 앞자리에 '0'을 채움
표시 문구"선택한예를 들어, 이 값을 5로 설정할 경우 숫자 앞에 0이 붙어 00001, 00002와 같은 5자리 숫자로 생성됩니다."
변경 적용 시점중간에 케이스 자릿수 변경 시, 이후 생성되는 케이스부터 적용

시작 번호 설정

매뉴얼 관리에서 자동 채번으로 전환하거나, 내부 차수 관리가 필요한 경우 번호의 시작점을 지정

항목내용
기본값1 (1 입력 시 1부터 시작)
설정 필드명Start From (시작 번호 입력란)
작동 방식사용자가 N을 입력하면 N부터 카운팅 시작. 예: 시작 번호 25 + 5자리 → 첫 Case ID는 00025
적용 시점설정 저장 후 생성되는 첫 번째 케이스부터 즉시 적용

최후 번호 검증 규칙

케이스 생성 시점→시작 번호는 해당 조합에서 이미 생성된 가장 최후 번호보다 커야 함
예시→3자리 설정, 시작 번호 003 설정, 기존 최후 번호 006이면 → 007부터 생성

검증 범위: 동일 조합 내에서 생성된 이력에 한함

시작 번호 설정 제한 정책

임계치 제한→시작 번호는 [최대 생성 가능 숫자 - 400]을 초과하여 설정할 수 없음

목적: 일련번호 고달(소진)을 방지하기 위한 하드웨어적 임계치 설정

안내 문구

"입력한 시작번호부터 보고자 관리번호가 생성됩니다. 예를 들어, 시작 번호를 '25'로 입력하고 자릿수를 '5자리'로 설정하면, 첫 번째 케이스 ID는 '00025'와 같은 형식으로 생성됩니다. 또한, 케이스 생성 시 기존 생성된 번호 이후의 숫자만 생성됩니다."

만료 임박 알림 팝업

항목내용
팝업 노출 조건남은 번호가 100개 이하일 때
노출 시점케이스에 Case ID가 최종적으로 부여된 순간 (Case ID Configuration 화면이 아님)
노출 대상해당 시스템을 사용하는 모든 사용자
버퍼 기준 사유시스템이 한 번에 최대 30개까지 병렬 처리 가능 → 충분한 버퍼로 100개 기준 설정
팝업 내용: "현재 마지막으로 부여된 케이스 ID는 [Case ID]입니다. 최대 자릿수[Case ID 최대값]에 도달하면 번호 체계가 자동으로 확장되어 [New Case ID_1], [New Case ID_2] 순으로 계속해서 부여됩니다. 번호 중복이나 작업 중단 없이 정상적으로 처리되니 안심하고 사용하시기 바랍니다."

팝업 닫기 로직

[Do not show again] 체크 + [Confirm]→해당 팝업을 더 이상 노출하지 않음
자동 미노출 조건→일련번호 완전 초과 후 새로운 자릿수 체계(1XXXXX...)로 넘어간 경우 기존 팝업 노출 중단

자동 자릿수 확장

설정된 자릿수를 모두 소진하더라도 서비스 중단 없이 시스템이 자동으로 자릿수를 확장하여 번호를 부여

확장 동작→최대 자릿수(예: 5자리) 초과 시 자릿수 한 자리 추가. '1XXXXX' 형식으로 번호 시작
구분값
