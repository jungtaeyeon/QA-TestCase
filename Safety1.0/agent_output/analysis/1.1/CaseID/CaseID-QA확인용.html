📁 분석 개요

기능 요약: ICSR(개별 사례 안전성 보고서) 식별 번호(Case ID) 생성 규칙을 구성하는 Admin 설정 기능.
번호 조합 구성요소(연도·보고구분·제품약어·자유텍스트·일련번호), 포맷 순서 설정, 자릿수, 시작 번호, Import 시 식별자 처리 방식, 만료 알림·자동 확장 등을 포괄함.

총 요구사항: 55건 (검증 대상: 54건 / 제외 TBD: 1건)
주요 리스크: Import 옵션 A와 C.1.8.1 자동 복사 로직 간 충돌(CONFLICT-002), Year=N/A 시 포맷 영역 비활성화 여부(CONFIRM-002)는 TC 작성 전 기획 확인 필수.

🔴 기획 충돌 (Conflict) — 2건
CONFLICT-001 🔴 충돌 영향도: High
Case Level 표시 형식 — "일련번호" vs "[설정된 번호 조합]" 불일치
문서 A Case ID의 개념
"해당 Case ID는 Case Edit의 Case ID에 표시되며 Tracker의 Case ID에 표시된다."
→ Case ID = [설정된 번호 조합] 전체 (Year + 보고구분 + 일련번호 등)
문서 B ICSR 식별자 관리 사양서 › Case vs Report Identifier 노출 사양
"Case Level | Tracker List, Case의 Case ID | 일련번호 | 예: 202604ST001"
→ 표시 형식 명칭이 "일련번호"이나 예시 202604ST001은 연도+구분자+번호 복합 조합
Tracker 및 Case Edit에 표시되는 값이 [설정된 번호 조합] 전체인지, 순수 일련번호만인지 불일치.
예시 202604ST001은 [설정된 번호 조합]처럼 보이나 형식 명칭은 "일련번호"로 기술됨.
권장 조치: ICSR 식별자 관리 사양서의 "일련번호" 명칭을 실제 표시 내용에 맞게 수정 필요. 기획 확인 후 단일 정의로 통일.
CONFLICT-002 🔴 충돌 영향도: High ⛔ TC 작성 보류
Import 옵션 A와 C.1.8.1 자동 복사 로직의 상호 충돌
문서 A Import 설정 › 옵션 A
"A. C.1.8.1만 유지 | 파일 내 값 사용 | C.1.8.1: 파일 내 데이터 유지"
→ 파일 내 C.1.8.1이 없는 경우 동작 미명시
문서 B C.1.1, C.1.8.1 동일 입력 설정
"C.1.8.1 값이 비어 있는 경우(Null)에 한해, C.1.1의 번호를 그대로 C.1.8.1에 복사하여 기입"
Import 옵션 A에서 파일 내 C.1.8.1이 Null인 경우, "C.1.8.1 파일 내 데이터 유지" 정책과 "Null이면 C.1.1 복사" 로직이 충돌함. 어느 로직이 우선 적용되는지 불명확.
권장 조치: Import 옵션 A + C.1.8.1 Null 케이스의 동작을 기획서에 명시. 기획 확인 전까지 Import TC 작성 보류.
🟠 표현 불일치 (Expression Gap) — 4건
EXPR-001 🟠 표현 불일치 영향도: Medium
"자유 텍스트" / "Free Text" / "Custom Text" 3종 혼용
표현 A"자유 텍스트"
섹션 제목 "자유 텍스트 추가", Chip 이름 "TXT=자유 텍스트"

"선택 시 [Custom Text] 블록 제공"
권장 조치: 세 표현 중 하나로 통일. 기획자 확인 후 결정.
EXPR-002 🟠 표현 불일치 영향도: Medium
"일련번호" / "시리얼 넘버" / "보고자 관리번호" 혼용
표현 A"일련번호"
ICSR 식별자 관리 사양서 Case Level 표시 형식 칼럼, 일련번호 자릿수 설정 섹션 제목
표현 B"시리얼 넘버"
일련번호 자릿수 설정 본문: "시리얼 넘버 길이를 5로 설정한 경우"
표현 C"보고자 관리번호"
안내 문구: "입력한 시작번호부터 보고자 관리번호가 생성됩니다"
권장 조치: 순수 일련번호를 지칭하는 표현을 하나로 통일 후 기획서 전체 적용.
EXPR-003 🟠 표현 불일치 영향도: Low
"보고 구분" / "Type of Report" / "TR" 혼용
표현 A섹션 제목
"Type of Report (보고 구분) 추가"
표현 BChip 이름
"TR=보고 구분"
권장 조치: 영/한 병기가 의도된 것이라면 기획서에 명시. 아니라면 통일 표현 지정.
EXPR-004 🟠 표현 불일치 영향도: Low
"Product Abbreviation" / "제품 약어" / "PRD" 혼용
표현 A섹션 제목
"Product Abbreviation 추가"
표현 B표시 문구
"Admin의 Product Config에 등록된 제품 약어를 참조합니다"
표현 CChip 이름
"PRD=제품 약어"
권장 조치: 의도적 병기라면 기획서에 명시. 아니라면 하나의 표현으로 통일.
🟡 누락 항목 (Missing) — 10건
MISSING-001 🟡 누락 영향도: High
Case ID Configuration 접근 권한/역할 미정의
Case ID 설정 화면은 Admin 기능으로 보이나, 어떤 역할(Role)이 조회/편집 가능한지 기술되지 않음. 권한 차이가 있는 사용자의 TC 작성 기준 불명확.
권장 조치: 역할별 접근 범위(조회/편집) 추가 기술 필요.
MISSING-002 🟡 누락 영향도: High
조직 식별자 입력 규칙 위반 시 에러 메시지 미정의
조직 식별자: 최소 1~최대 10글자, 영어·숫자만. 위반(11글자 이상, 특수문자 입력 등) 시 에러 메시지 문구 없음.
권장 조치: 에러 조건별 메시지 문구 추가 필요.
MISSING-003 🟡 누락 영향도: Medium
자유 텍스트 입력 규칙 위반 시 에러 메시지 미정의
자유 텍스트: 영문·숫자만, 최대 6글자. 위반 시 에러 메시지 없음.
권장 조치: 에러 조건별 메시지 문구 추가 필요.
MISSING-004 🟡 누락 영향도: Medium
시작 번호 설정 제한 초과 시 에러 메시지 미정의
시작 번호 ≤ [최대 생성 가능 숫자 - 400] 제한이 있으나, 초과 입력 시 에러 메시지 및 표시 시점 미기술.
권장 조치: 제한 초과 시 에러 메시지 문구 및 표시 시점(입력 중/저장 시) 추가 필요.
MISSING-005 🟡 누락 영향도: Medium

국가코드 우선순위 목록에서 3번 항목 하위에 "1. (First)" 항목이 중첩 기술되어 있어, 전체 우선순위가 4단계인지 5단계인지 불명확.
권장 조치: 우선순위를 명확한 1→N 순번으로 재기술 필요.
MISSING-006 🟡 누락 영향도: High

권장 조치: 일련번호 칩의 필수 여부 명시 필요. (필수 포함/삭제 불가 vs 선택적 추가)
