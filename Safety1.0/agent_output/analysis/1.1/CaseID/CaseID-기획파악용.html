📌 기능 개요
목적

ICSR(개별 사례 안전성 보고서)의 식별 번호 생성 규칙을 표준화하여 데이터 정합성을 확보하고 관리 효율성을 증대함.


대상 필드

Case ID 필드. 추후 국가코드, 조직명을 포함하여 C.1.1 R3 필드에 매핑.


필수 전제 규칙
  • 설정 가능한 모든 텍스트는 영문 및 숫자만 가능, 대소문자 구분하여 입력 가능
  • ID 재사용 방지: 한 번 부여된 일련번호는 케이스가 삭제되더라도 절대 재사용되지 않음

  • 전체 형식: [국가코드]-[조직 식별자]-[설정된 번호 조합]
  • Case ID = [설정된 번호 조합] 부분만을 의미함
  • 표시 위치: Case Edit의 Case ID 필드, Tracker의 Case ID 열

⚙️ 번호 조합 구성요소 설정

Year, Type of Report, Product Abbreviation, 자유 텍스트, 일련번호의 조합으로 생성됨

1. Year (연도)

최초 발생인지일(C.1.4)의 데이터값 참조


구분표기 방식번호 초기화
2-digit yearYY (예: 26)매년 1월 1일 00:00:00 리셋
4-digit yearYYYY (예: 2026)매년 1월 1일 00:00:00 리셋
N/A연도 미표기누적 증가 (초기화 없음)

툴팁 문구: "연도 선택 시 매년 초, 연월 선택 시 매월 초에 일련번호가 '00001'로 초기화됩니다. N/A 선택 시(Chip을 추가하지 않았을 경우) 초기화 없이 번호가 계속 누적됩니다."

2. Type of Report (보고 구분)

체크 시 케이스의 Type of Report 값에 따라 구분자가 Case ID에 자동 추가


Type of Report 값추가 코드
SpontaneousSP
Report from StudyRS
OtherOT
Not Available to Sender (Unknown)UN

표시 문구: "체크 시 케이스의 Type of Report 값에 따라 구분자가 Case ID에 자동 추가됩니다."

Preview 표시: SP

3. Product Abbreviation (제품 약어)
  • 데이터 소스: Admin 내 Product Config에 등록된 Product Abbreviation(제품 약어) 필드값 참조
  • 노출 조건: 케이스에 '제품 불러오기'로 제품 정보가 포함된 경우에만 활성화. 값이 없으면 UNK 표시
  • Suspect이자 자사 제품이 2개 이상인 경우: 첫 번째 순서의 자사 제품 약어 표시
  • Preview 표시: PRD

표시 문구: "Admin의 Product Config에 등록된 제품 약어를 참조합니다. 값이 없으면 UNK으로 표시됩니다."

4. 자유 텍스트 (Free Text / Custom Text)
  • 체크박스 형태로 선택 가능
  • 선택 시 [Custom Text] 블록 제공
  • 라디오 버튼 선택하여 텍스트 블록 추가
  • 사용자가 원하는 문자(영문, 숫자만) 직접 입력, 최대 6글자
  • 입력된 텍스트는 드래그 앤 드롭으로 원하는 위치에 배치 가능
🔢 일련번호 자릿수 및 시작점 설정
일련번호 자릿수 설정
  • 선택값: 3, 4, 5, 6, 7, 8, 9 (최대 9자리)
  • 선택한 자릿수만큼 번호 앞자리에 '0'을 채움(Zero-padding)
  • 중간에 자릿수 변경 시, 이후 생성되는 케이스부터 적용

표시 문구: "선택한 자릿수를 5로 설정할 경우 숫자 앞에 0이 붙어 00001, 00002와 같은 5자리 숫자로 생성됩니다."

시작 번호 설정
  • 기본값: 1 (1 입력 시 1부터 시작)
  • 사용자가 숫자 N을 입력하면 N부터 카운팅 시작
  • 적용 시점: 설정 저장 후 생성되는 첫 번째 케이스부터 즉시 적용

최후 번호 검증 규칙
  • 시작 번호는 해당 조합에서 이미 생성된 가장 최후 번호보다 커야 함
  • 예: 최후 번호가 006이면 → 007로 생성됨
  • 검증 범위: 동일 조합 내 생성 이력에 한함

시작 번호 설정 제한
  • 제약: 시작 번호는 [최대 생성 가능 숫자 - 400]을 초과하여 설정할 수 없음

안내 문구: "입력한 시작번호부터 보고자 관리번호가 생성됩니다. 시작 번호를 '25'로 입력하고 자릿수를 '5자리'로 설정하면, 첫 번째 케이스 ID는 '00025'와 같은 형식으로 생성됩니다."

🖱️ 포맷 설정 및 인터랙션 (Drag & Drop)
  • 노출 조건: Year를 연도(2/4-digit)로 선택하거나, Type of Report를 체크할 경우에만 활성화
  • Chip 형태의 블록으로 구성: YY/YYYY, ######(일련번호), TR(보고 구분), PRD(제품 약어), TXT(자유 텍스트)
  • Chip 추가 시 하단의 해당 설정 영역 활성화
  • Chip 드래그 앤 드롭으로 좌우 순서 변경 → 상단 Preview에 실시간 반영
  • x 클릭 시 해당 칩 삭제, 하단 영역 비활성화 (이전 설정값은 유지)

표시 문구: "칩을 드래그하여 순서를 변경하면 상단 Preview에 실시간 반영됩니다."

예시: YYYY - ##### - TR 순서 배치 → KR-SELTA-202500001TR
